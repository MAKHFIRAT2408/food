О проекте

Данный проект представляет собой RESTful API-приложение, предназначенное для автоматизации процессов онлайн-доставки еды.
Система реализована с использованием фреймворка FastAPI и обеспечивает взаимодействие между пользователями, ресторанами и сервисом оформления заказов.

Приложение ориентировано на использование в качестве серверной части веб- или мобильного приложения службы доставки еды.

Основной функционал

Приложение предоставляет следующие возможности:

регистрация и аутентификация пользователей;

разграничение прав доступа (пользователь / администратор);

управление ресторанами и меню (создание, редактирование, просмотр);

просмотр списка доступных блюд;

оформление заказов на доставку еды;

просмотр истории заказов пользователя;

автоматическая обработка данных через REST API;

доступ к интерактивной документации API.

Используемые технологии

В процессе разработки были использованы следующие технологии и инструменты:

FastAPI — основной фреймворк для разработки API;

SQLAlchemy — ORM для работы с базой данных;

SQLite — система управления базами данных;

Pydantic — валидация и сериализация данных;

JWT (JSON Web Token) — механизм аутентификации и авторизации;

Uvicorn — ASGI-сервер для запуска приложения;

Pytest — модульное тестирование API.

Возможности API
Аутентификация

POST /auth/register — регистрация пользователя

POST /auth/login — вход в систему и получение JWT-токена

Пользователи

GET /users/me — получение информации о текущем пользователе

Рестораны

GET /restaurants/ — просмотр списка ресторанов

POST /restaurants/ — создание ресторана (только для администратора)

Блюда

GET /dishes/ — просмотр меню

POST /dishes/ — добавление блюда (только для администратора)

Заказы

POST /orders/ — оформление заказа

GET /orders/ — просмотр заказов пользователя

DELETE /orders/{id} — отмена заказа

Как назначить себя администратором

После первой регистрации пользователя:

Запустите приложение;

Зарегистрируйтесь через эндпоинт /auth/register;

Остановите сервер;

Откройте файл базы данных food_delivery.db в программе DB Browser for SQLite;

Перейдите во вкладку Browse Data и выберите таблицу users;

Найдите свою запись и измените значение поля is_admin с 0 на 1;

Сохраните изменения и перезапустите сервер.

После этого пользователь будет обладать правами администратора.

Как запустить проект локально
Требования

Python 3.10 или выше

Git

(опционально) DB Browser for SQLite

Пошаговая инструкция

Клонировать репозиторий:

git clone https://github.com/ВАШ_ЛОГИН/food-delivery-api.git
cd food-delivery-api


Создать виртуальное окружение:

python -m venv venv


Активировать виртуальное окружение:

Windows

venv\Scripts\activate


Linux / macOS

source venv/bin/activate


Установить зависимости:

pip install -r requirements.txt


Запустить сервер:

uvicorn main:app --reload


Приложение будет доступно по адресу:
http://127.0.0.1:8000
